# 변경 이력 (Changelog)

## 버전 규칙

3Ls는 **MAJOR.MINOR** 두 자리 버전을 씁니다.

| 자리 | 올리는 기준 | 예시 |
|---|---|---|
| **MAJOR** | 새 언어팩/콘텐츠 데이터셋 추가 (학습팩이든 사전팩이든) | 프랑스어→+스페인어 = v1→v2 |
| **MINOR** | 새 언어팩 없이 UI·사용성·기능 개선만 있을 때. **MAJOR가 오르면 0으로 리셋** | 다크모드 추가 = v3.0→v3.1 |

- 새 버전을 낼 때는 **`index.html`의 `APP_VERSION`** 과 **`service-worker.js`의 `CACHE`** 문자열을 반드시 함께 올립니다(하나만 바꾸면 화면에 보이는 버전과 실제 캐시가 어긋납니다).
- `manifest.json`의 `version` 필드는 브라우저가 읽지 않는 참고용 표기이며, 위 둘과 같은 값으로 맞춰둡니다.
- 버그 수정처럼 아주 사소한 변경까지 세분화할 필요가 생기면 그때 PATCH(x.y.z) 자리를 추가하는 것으로 확장 가능(현재는 미사용).
