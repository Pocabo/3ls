# 변경 이력 (Changelog)

## 버전 규칙

3Ls는 **LANG.MAJOR.MINOR.PATCH** 네 자리 버전을 씁니다. **LANG 자리만 봐도 지원 언어 개수를 알 수 있게** 설계했습니다.

| 자리 | 올리는 기준 | 값의 의미 / 예시 |
|---|---|---|
| **LANG** | **새 언어**를 추가 | 값 = 지원 언어 개수. 현재 4 (프랑스어·스페인어·일본어·한자어) |
| **MAJOR** | 대형 기능/**새 콘텐츠 축**이 생김 — 앱 전체에 걸친 기능 메뉴, 또는 "English→French" 같은 새 학습 방향. **LANG이 오르면 0으로 리셋** | 📚 단어은행 신설 |
| **MINOR** | 메뉴 추가 없이 **UI가 바뀌어 UX가 달라짐**. **MAJOR가 오르면 0으로 리셋** | 다크모드, 열 접기, 어절 단위 줄바꿈, Contents 드롭다운·언어 선택 페이지 |
| **PATCH** | UI 오류·버그 수정, 또는 **기존 언어에 콘텐츠/단어 추가**. **MINOR가 오르면 0으로 리셋** | 발음 열 줄바꿈 수정, 한자어 단어 추가, `fr-extra` 확장 팩 추가 |

- **왼쪽 자리가 오르면 그 오른쪽 자리는 모두 0으로 리셋**합니다 (예: 새 언어 추가 → `x.0.0.0`).
- 기존 언어의 **확장/사전 팩**(예: `fr-extra`, `parent:'fr'`)은 새 언어가 아니므로 LANG·MAJOR를 올리지 않습니다. 언어 선택 화면엔 뜨지 않고, 상단 **📂 Contents** 드롭다운에서 접근합니다.
- **⚠️ 팩 규격 변경**: 언어팩 데이터 구조(필드 추가/의미 변경, 예: `tip` 필드 도입, `parent`/`menuLabel` 필드, 열 자동 숨김 규칙)가 바뀌면 자리와 별개로 해당 항목에 `⚠️ 팩 규격 변경` 을 표기해, 나중에 팩을 만들 때 맞춰야 할 규격을 한눈에 보이게 합니다.
- 참고: v4.1.0.0 이전 항목은 옛 2~3자리 번호 체계로 기록돼 있습니다(역사 보존용). 아래 v4.1.0.0부터 이 4자리 규칙을 적용합니다.
- 새 버전을 낼 때는 **`index.html`의 `APP_VERSION`**, **`service-worker.js`의 `CACHE`**, **`manifest.json`의 `version`**, **`README.md`의 현재 버전** 을 모두 같은 값으로 맞춥니다(하나만 바꾸면 화면 표기·캐시·문서가 어긋납니다).
- `manifest.json`의 `version` 은 브라우저가 읽지 않는 참고용 표기입니다.

## v4.1.0.0 — 2026-07-16
**언어 선택 페이지 + 📂 Contents 드롭다운 · 헤더 라벨 영어화 · 4자리 버전 체계 확정** ⚠️ 팩 규격 변경
- **언어 선택 페이지 신설**: 헤더의 언어 `<select>` 드롭다운을 제거하고, 언어를 카드로 고르는 별도 화면(`picker`)으로 분리 — 언어가 늘어도 드롭다운이 넘치지 않음
- **📂 Contents 드롭다운**: 현재 언어의 콘텐츠(본과정 + 확장/사전 팩)를 전환하고, "🌐 언어 선택…"으로 선택 페이지 진입. 기존 언어 드롭다운을 대체
- **`fr-extra` 확장 팩을 프랑스어 하위로 이동**: `parent:'fr'` 지정 → 언어 선택엔 안 뜨고 프랑스어의 Contents에서 접근. `⚠️ 팩 규격 변경`: 팩에 선택적 `parent`(상위 언어 id)·`menuLabel`(드롭다운 표기) 필드 추가
- **헤더 라벨 영어화**: 단어은행→**word**, 복습→**remind**, 메뉴→**menu**, 새 버튼 **Contents** (본문·푸터 등 나머지 한국어 유지 — 현재는 "한국어→타 언어" 구성)
- **버전 체계를 `LANG.MAJOR.MINOR.PATCH`로 확정**(위 규칙표): LANG=언어 수(4), MAJOR=대형 기능(단어은행=1). 직전 임시 `v5.0.0`/`v4.2.0.0` 표기를 이 기준으로 **v4.1.0.0**으로 재정렬

## v4.3.1 — 2026-07-10
**PATCH · 표 열 접기 + 팁 숨김**
- 표 위 "열 접기" 칩으로 원하는 열(분류·발음·뜻·예문·듣기)을 접었다 펼 수 있음 (단어 열은 항상 표시)
- 뜻 열 안의 **💡팁**도 별도 칩으로 숨김 가능
- 선택은 localStorage에 저장되어 새로고침·재접속 후에도 유지. 데이터에 없는 열/팁은 칩에도 표시되지 않음

## v4.3.0 — 2026-07-10
**MINOR · 한국어 줄바꿈을 어절 단위로**
- `word-break: keep-all` 적용 → 한국어 텍스트가 글자 단위가 아니라 **어절(공백) 단위로만** 줄바꿈됨 (단어가 중간에 끊기던 문제 해결)
- 표·카드·설명 등 앱 전역에 적용. 라틴 문자(프랑스어/스페인어)는 기존대로 공백 단위로 끊기고, 페이지 가로 오버플로우 없음(표만 스크롤)

## v4.2.0 — 2026-07-10
**MINOR · 표를 데이터 주도로 (호환성 강화)**
- 언어팩 데이터에 **값이 하나도 없는 필드는 해당 열 자체를 렌더하지 않음**
  (예: 한자어 팩처럼 `ex`/`exKo`가 모두 `''`이면 "예문" 열이 사라짐)
- 발음(`ipa`), 뜻(`ko`/`en`/`tip`), 예문(`ex`) 열 모두 동일 규칙 적용 — 새 언어팩이 어떤 필드를 채우든 표가 자동으로 맞춰짐
- 레벨 단어 표와 단어은행 표가 공통 렌더러(`wordTableHtml`)를 쓰도록 통합
- 열 개수에 맞춰 표 최소 너비를 계산 → 열이 적은 팩은 모바일에서 불필요한 가로 스크롤이 생기지 않음
- 카드 모드에서도 빈 `ko`/`exKo`가 빈 줄로 남지 않도록 정리
