# 3Ls — Language Learning Library

CEFR 기반 10레벨 커리큘럼으로 언어를 학습하는 **설치형 웹앱(PWA)** 입니다.
서버·앱스토어·APK 불필요 — 브라우저로 열거나, 홈 화면에 설치해 **오프라인**으로 쓸 수 있습니다.

**현재 버전: v5.2.0.0** — 변경 이력과 버전 규칙은 [CHANGELOG.md](CHANGELOG.md) 참고.
> 요약(4자리 `LANG.MAJOR.MINOR.PATCH`): **LANG**=지원 언어 수, **MAJOR**=대형 기능/새 콘텐츠 축(단어은행·새 번역 방향 등), **MINOR**=UI·UX 변경, **PATCH**=오류 수정·기존 팩 단어 추가. 왼쪽이 오르면 오른쪽은 0으로 리셋.
> LANG 자리만 봐도 지원 언어 개수를 알 수 있습니다. 새 버전을 낼 땐 `index.html`의 `APP_VERSION`, `service-worker.js`의 `CACHE`, `manifest.json`의 `version`, 이 문서의 현재 버전을 함께 올려주세요.

## 커리큘럼 구조 (프랑스어 🇫🇷 기본 탑재)

| 레벨 | CEFR | 단계 | 내용 |
|---|---|---|---|
| 1-2 | A1.1 | 기초 다지기 | 발음 규칙(리에종·묵음), 인사, 숫자, **관사·인칭대명사·부정문·의문사**¹ |
| 3-5 | A1-A2 | **Phase A** 블록 조립 | 음식·가족·날씨·일과 어휘, 성수일치, 대명동사 |
| 6-8 | B1 | **Phase B** 빈칸/변형 | passé composé, imparfait, 미래시제, 목적어 대명사 y/en |
| 9-10 | B2-C1 | **Phase C** 자유 작문 | subjonctif, conditionnel, 문체(격식) |

¹ 원 커리큘럼에 없던 문법 연결부(관사, 대명사, 부분관사, 소유형용사 등)를 레벨 사이 징검다리로 추가했습니다.

### 핵심 기능
- **📚 단어은행**: 헤더 버튼 하나로 현재 언어팩의 **모든 레벨/분류 단어를 한 화면에서** 검색·필터 (Notion 데이터베이스 뷰의 대체재)
- **Library 모드**: 레벨 잠금 없이 모든 단계를 자유 열람. 시험은 선택 연습(80점 기준은 자기 점검용)
- **Base→Target**: 설명 언어(base)를 한국어/English로 전환하면 UI와 표시되는 뜻이 함께 바뀜 (예: KR→FR, EN→FR)
- **누적 복습 루프(SRS)**: 플래시카드에서 "몰라요"한 단어가 0→1→3→7→14→30일 간격으로 재출제
- **오디오-비주얼 그라운딩**: 전 단어 IPA 발음기호 + 브라우저 TTS(🔊) 지원
- **특수문자 키보드**: é è ç 등 언어별 입력 버튼 자동 제공
- **다크/라이트 모드**: ⚙ 메뉴에서 시스템·라이트·다크 순환 전환 (선택은 기기에 저장)
- 표는 가로 스크롤 지원 — 열이 많아도 모바일 화면 밖으로 벗어나지 않음
- 진행 상황은 브라우저 localStorage에 언어별로 저장

### 📚 단어은행 사용법
헤더의 **📚 단어은행**(모바일에선 아이콘만)을 누르면 레벨 카드를 일일이 열지 않고
**단어·뜻·예문·발음(IPA)** 전체를 한 표에서 검색·필터할 수 있습니다.
- 검색창: 단어/뜻/예문/발음 어디든 부분 일치로 실시간 필터
- 칩 필터: 레벨(학습팩) 또는 분류(사전팩)별로 좁혀보기
- 사전 전용 팩(`dictionary:true`, 예: 일본 음식)도 동일하게 지원

## 🌍 다른 언어 추가하기 (호환성 설계)

엔진(`index.html`)은 언어를 전혀 모릅니다. 모든 언어 지식은 **언어팩**이 공급합니다.

```
3Ls/
├── index.html          ← 학습 엔진
├── manifest.json       ← PWA 매니페스트
├── service-worker.js   ← 오프라인 캐시 (언어팩 추가 시 APP_SHELL에도 등록)
├── icon.svg            ← 앱 아이콘 (+ icon-maskable.svg)
├── .nojekyll           ← GitHub Pages Jekyll 비활성화
└── languages/           ← 파일명 = target 기준 (한 파일이 여러 base 뜻을 담음)
    ├── fr.js            ← 프랑스어 (본과정, m:{kr,en})
    ├── fr-extra.js      ← 프랑스어 확장 사전 (extra:true)
    ├── es.js            ← 스페인어
    ├── ja.js            ← 일본어 (빈 껍데기 → 사전으로 redirect)
    ├── ja-food.js       ← 일본 음식 사전 (ja 하위 extra)
    ├── kr.js            ← 한국어 (빈 껍데기)
    ├── kr-hanja.js      ← 한자어 사전 (kr 하위 extra)
    └── _template.js     ← 새 언어팩 작성 가이드
```

> 새 언어팩을 추가하면 ① `index.html` 하단에 `<script>` 한 줄, ② `service-worker.js`의
> `APP_SHELL` 배열에 경로를 넣어야 오프라인에서도 로드됩니다.

### 🌐 base(설명 언어) · target(학습 언어)
- 팩은 **`target`(배우는 언어)만** 선언합니다. 지원하는 **base(설명 언어)는 단어 `m`의 키로 자동 결정**돼요(예: `m:{kr,en}` → KR·EN 둘 다 지원).
- 언어 선택 화면 상단의 **Base 드롭다운**으로 base를 바꾸면 **UI 언어**와 **표시되는 뜻**이 함께 바뀝니다(KR 모드=한국어 뜻만, EN 모드=영어 뜻만). 현재 base로 뜻이 없는 target은 카드에 안 뜹니다.
- **확장 팩**: 같은 `target`에 `extra:true` 팩을 두면 언어 선택 카드의 **▾** 와 📂 Contents 드롭다운에서 뜹니다. 표시명은 `menuLabel`. 본과정이 빈(levels:[]) target은 카드 클릭 시 1순위 확장 사전으로 redirect.
- **나중에 base 추가**: 각 단어 `m`에 키를 더하거나(A), `registerMeanings('fr','ja',{ 'bonjour':'こんにちは', … })` 훅으로 별도 뜻 파일을 얹을 수 있습니다(B, 표제어키 오버레이).

**방법 1 — JS 파일**: `_template.js`를 복사해 `target`을 채우고, `index.html` 하단에 `<script>` 한 줄 추가.
**방법 2 — JSON 가져오기**: 팩 객체를 `.json`으로 저장 후 ⚙ 메뉴의 **언어팩 가져오기**로 불러오기(브라우저에 저장됨).

성(性)이 없는 언어(일본어 등)는 `gender` 필드를 생략하면 UI가 자동으로 숨겨지고, `tts` 로케일만 바꾸면 발음도 그 언어로 나옵니다.

### 🧩 필요한 필드만 채우세요 (열 자동 숨김)
표는 **데이터 주도**입니다. 어떤 필드가 팩 전체에서 비어 있으면(`''`·`undefined`) **그 열은 아예 렌더되지 않습니다.**

| 필드 | 없으면 |
|---|---|
| `ipa` | "발음" 열 숨김 |
| `m`(현재 base 뜻) / `tip` (둘 다 없을 때) | "뜻" 열 숨김 |
| `ex` / `exm` | "예문" 열 숨김 |
| `gender` | 성별 태그 숨김 |

예) 한자어 사전은 `ex`·`exKo`를 모두 `''`로 둬서 "예문" 열이 표시되지 않습니다.
`단어`와 `듣기` 열만 항상 표시되므로, **최소한 `w`(표제어)만 있어도 동작**합니다.
